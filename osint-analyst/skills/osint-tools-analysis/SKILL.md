## Доступность инструментов (analysis-блок)

| Инструмент | Тип доступа | Как проверить | Если недоступен |
|------------|------------|---------------|-----------------|
| **ExifTool** | CLI | `which exiftool` | Используй online EXIF viewer через webfetch |
| **VirusTotal** | Web API | `VT_API_KEY` в env | Используй urlscan.io (бесплатно) |
| **urlscan.io** | Web (public) | Всегда доступен | — |
| **FOCA, Metagoofil** | Desktop/CLI | Требуют установки | Пропусти; предложи ручной шаг |
| **Google/Яндекс дорки** | Web (public) | Всегда доступны через webfetch | — |

**Правило**: если инструмент недоступен — **не симулируй его использование**. Зафиксируй в «Пробелы», выбери альтернативу или предложи пользователю ручной шаг.

## 🛠 Блок 7 — Анализ файлов и метаданных

| Инструмент  | Назначение                                  | Метод |
|-------------|---------------------------------------------|-------|
| ExifTool    | EXIF-метаданные изображений и документов    | White |
| FOCA        | Метаданные документов (PDF, DOCX, XLSX)     | White |
| Metagoofil  | Автосбор метаданных через Google            | White |
| VirusTotal  | Анализ файлов, URL, IP, хэшей (70+ движков) | White |
| urlscan.io  | Поведенческий анализ URL без прямого посещения | White |

**Когда применять:**
- ExifTool / FOCA — при наличии документов от объекта (фото, PDF, Word)
- VirusTotal — при проверке инфраструктуры (домен, IP, файл)
- urlscan.io — безопасный анализ подозрительных ссылок
- Metagoofil — автоматический сбор документов с сайта организации

## 🔍 Блок 8 — Поисковые операторы и дорки

| Платформа     | Ключевые операторы                                    | Специфика                          |
|---------------|-------------------------------------------------------|------------------------------------|
| Google Dorks  | `site:`, `filetype:`, `intitle:`, `inurl:`, `cache:` | Глобальный охват                   |
| Яндекс Дорки  | `host:`, `mime:`, `lang:`, `date:`                    | СНГ глубже; кириллица, госсайты    |
| Bing Dorks    | `ip:`, `site:`, `feed:`                               | Уникальная индексация, резервный   |
| GitHub Dorks  | `filename:`, `extension:`, `content:`                 | Секреты, API-ключи, токены в коде  |

**Шаблоны дорков для СНГ:**

**Яндекс — поиск документов на сайте организации:**
```
host:company.by filetype:pdf
```

**Google — утечки конфигов:**
```
site:pastebin.com "company.ru" password
```

**GitHub — API-ключи компании:**
```
"company.by" filename:.env
```

**Яндекс — сотрудники в LinkedIn/HH:**
```
"ООО Название" site:hh.ru OR site:linkedin.com
```

## 🐍 Блок 9 — OSINT-библиотеки Python (osrframework)

| Инструмент   | Назначение                                   | Метод |
|-------------|----------------------------------------------|-------|
| osrframework | Набор CLI-утилит: `mailfy.py`, `usufy.py`, `phonefy.py`, `searchfy.py`, `domainfy.py` | White |

**Когда применять:**
- `mailfy.py` — поиск email в утечках и сервисах
- `usufy.py` — верификация username на 300+ платформах
- `phonefy.py` — разведка по номеру телефона
- `searchfy.py` — мультиплатформенный поиск профилей
- `domainfy.py` — базовая разведка домена

**Если недоступен:**
- Каждая утилита имеет аналог в других навыках (usufy → Sherlock, phonefy → phoneinfoga, domainfy → amass)

## Яндекс-специфичные операторы (СНГ)

При работе с объектами из СНГ Яндекс — приоритетная поисковая система. Используй эти операторы ДО Google Dorks для СНГ-объектов.

### Операторы Яндекса

| Оператор | Назначение | Пример |
|----------|------------|--------|
| `host:` | Поиск только на указанном сайте (аналог site:) | `host:company.by` |
| `mime:` | Тип документа (pdf, doc, xls, ppt, rtf) | `host:company.by mime:pdf` |
| `lang:` | Язык документа (ru, en, uk, be) | `"отчётность" lang:ru` |
| `date:` | Дата публикации (YYYYMMDD) | `date:2024*` — все документы за 2024 год |

### Шаблоны дорков для Яндекса (СНГ)

**Документы организации:**
```
host:company.by mime:pdf
```

**Кириллица + дата (персоны):**
```
"Иванов Иван Иванович" date:2024*
```

**Поиск по госсайтам РФ:**
```
host:nalog.gov.ru "ИНН 7707083893"
```

**Поиск по региональным форумам (Беларусь):**
```
host:onliner.by "никнейм" date:2025*
```

**Поиск упоминаний компании в новостях:**
```
"ООО Название" host:yandex.ru/news
```

**Поиск email на домене (Яндекс):**
```
host:company.by "@company.by"
```

**Поиск по LinkedIn/HH.ru (сотрудники):**
```
"ООО Название" host:hh.ru OR host:linkedin.com
```

### Приоритет поисковых систем для СНГ

| Задача | Приоритетная система | Причина |
|--------|---------------------|---------|
| Поиск персон (кириллица) | Яндекс | Лучше индексирует RuNet, VK, OK |
| Поиск документов организаций | Яндекс | `mime:` и `host:` точнее чем Google filetype: |
| Поиск утечек/секретов | Google + GitHub | Google индексирует Pastebin, GitHub |
| Поиск по email (международный) | Google | Шире охват за пределами СНГ |
| Поиск субдоменов | crt.sh / Amass (не поисковик) | SSL-сертификаты точнее чем дорки |
