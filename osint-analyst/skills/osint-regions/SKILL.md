## Региональная специализация

### СНГ (Россия, Беларусь, Украина, Казахстан, Узбекистан, Молдова, Армения, Грузия)

**Поисковые операторы:**
- Яндекс, Mail.ru, Rambler (кириллица + транслит)

**Соцсети:**
- ВКонтакте, Одноклассники, TenChat, Telegram

**Госреестры:**
- egrul.nalog.gov.ru, egr.gov.by, data.gov.ua, portal.nalog.gov.by
- kad.arbitr.ru, суды.рф, fssp.gov.ru, Росреестр, ГИБДД

**Утечки СНГ:**
- Понимаешь структуру дампов (кодировка, формат), умеешь коррелировать данные из утечек

**Telegram-боты для разведки:**
- Глаз Бога, Quick OSINT, TeleSINT и др.
- Все — Grey, кросс-верификация обязательна
- См. `osint-tools-grey` для полного списка

**Агрегаторы и «пробивные» сервисы:**
- Понимаешь источники, ограничения и ошибки
- Достоверность: 60–80%
- Правовой риск: HIGH (ФЗ-152 РФ, Закон № 99-З РБ)

---

### Европа (ЕС + EEA + Великобритания)

**Реестры компаний:**
- Companies House (UK), Handelsregister (DE), KRS (PL)
- Infogreffe (FR), e-Justice (EU), OpenCorporates

**Кадастры:**
- HM Land Registry (UK), Grundbuch (DE), Cadastre (FR)

**Санкционные базы:**
- OFAC, EU Sanctions List, OpenSanctions, FinCEN

**Правовая рамка — GDPR:**
- Явно указывай ограничения при работе с данными граждан ЕС
- Минимизация данных: собирай только необходимое для цели исследования
- Срок хранения: временный, удаляй после завершения задачи (если не оговорено иное)
- Transborder data transfer: фиксируй, если данные пересекают границы ЕС

## Decision Tree: Беларусь (обязательный правовой gating)

> ⚠️ **Применяй до начала сбора данных**, если юрисдикция объекта — Республика Беларусь.
> Источник: методология OSINT для СНГ, Часть II, Приложение.

Перед запуском любого инструмента для объекта в юрисдикции РБ пройди дерево решений:

```
Ресурс в списке экстремистских материалов РБ?
   ДА → СТОП. Сообщи пользователю:
         «Объект или источник находится в „Реестре экстремистских материалов“ РБ.
          Подписка/доступ — ст. 19.11 КоАП (штраф/арест); пересылка — ст. 361-1/361-4 УК РБ (до 7 лет).
          Исследование невозможно легальными методами.»
         Зафиксируй в «Пробелах»: «Объект в реестре экстремистских материалов РБ — исследование остановлено».
   НЕТ ↓

Это персональные данные физлица (ФИО + контакт)?
   ДА → Есть согласие субъекта (Закон № 99-З)?
         НЕТ → СТОП. «Сбор ПД физлица без согласия — ст. 203-1 УК РБ (до 5 лет лишения свободы).»
         ДА → Продолжить с маркировкой «Согласие получено».
   НЕТ ↓

Данные получены обходом защиты/паролей?
   ДА → СТОП. «Несанкционированный доступ — ст. 352/349 УК РБ.»
   НЕТ ↓

Объект — должностное лицо, силовик, судья, чиновник?
   ДА → СТОП. «Высокий правовой риск. Рекомендую воздержаться от исследования.»
   НЕТ ↓

ОК. Работаем аккуратно, документируем, ссылаемся только
на открытые источники. В договоре: «Информационно-аналитические услуги»
(НЕ «Аудит ИБ» — требуется лицензия ОАЦ, Закон № 213-З).
```

### Правила ROE (Rules of Engagement) для РБ

| Правило | Основание |
|---------|-----------|
| Явный запрет на сбор ПД физлиц без согласия | Закон № 99-З; ст. 203-1 УК РБ |
| Исключение объектов, связанных с госаппаратом и силовиками | Высокий риск |
| Не использовать «Аудит ИБ» в документах — только «Информационно-аналитические услуги» | Закон № 213-З (лицензия ОАЦ) |
| При встрече с экстремистским ресурсом: закрыть вкладку, очистить кэш, без скриншотов | Ст. 19.11 КоАП / ст. 361-1 УК РБ |
| Использовать VPN/IP, не ассоциированные с офисом | Мониторинг VPN-трафика в РБ |

## Идентификаторы СНГ — ключи для поиска в реестрах

Знание национальных идентификаторов критично для поиска в местных реестрах. Используй их как поисковые ключи при работе с государственными базами данных.

| Страна | Физлицо | Юрлицо | Авто | Примечание |
|--------|---------|--------|------|------------|
| 🇷🇺 Россия | ИНН (12 цифр) | ИНН (10 цифр) / ОГРН (13 цифр) | ГИБДД-номер | ИНН физлица можно проверить на nalog.gov.ru |
| 🇧🇾 Беларусь | — (ПД, сбор ограничен) | УНП (9 цифр) | Рег. номер | УНП — основной ключ для ЕГР (egr.gov.by) |
| 🇺🇦 Украина | РНОКПП (ИНН, 10 цифр) | ЕДРПОУ (8 цифр) | Рег. номер | РНОКПП = ИНН; поиск через data.gov.ua |
| 🇰🇿 Казахстан | ИИН (12 цифр) | БИН (12 цифр) | ГРНЗ | ИИН/БИН — ключи для реестра ГКС |
| 🇺🇿 Узбекистан | ИНН (физлица) | ИНН (юрлица) | ГРНЗ | Реестр: registr.stat.uz |

**Алгоритм использования идентификаторов:**
1. При поиске юрлица: сначала найди ИНН/УНП/БИН/ЕДРПОУ → затем ищи в соответствующем реестре.
2. При поиске физлица (РФ/KZ): ИНН/ИИН → проверка через налоговую + ФССП + КАД.
3. При поиске авто: госномер → ГИБДД (РФ) / @GtaSearchOsntBot (Grey) / @HimeraLeakbot (Grey).
4. Для кросс-верификации: один и тот же идентификатор ищи в ≥2 реестрах.
