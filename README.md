# MAS — Multi-Agent System

Набор готовых **мультиагентных конфигураций** для агентского ПО. Каждая система — это профиль под свою задачу: подобранные роли агентов, их навыки (skills), модель прав и правила взаимодействия. Профили независимы друг от друга — берите тот, который нужен под конкретную работу, и разворачивайте его.

Сейчас в репозитории две системы:

| Система | Назначение | Каталог |
|---|---|---|
| **Команда разработки** | Полный конвейер разработки ПО — от исследования кодовой базы до ревью | [`development-team/`](development-team/) |
| **OSINT-аналитик** | Разведка и поиск информации по открытым и полуоткрытым источникам | [`osint-analyst/`](osint-analyst/) |

---

## Системы

### 1. Команда разработки (`development-team/`)

Иерархическая мультиагентная система (orchestrator-worker), построенная по принципам **loop engineering** с человеко-гейтнутым, аудируемым циклом. Координирующий агент `team-lead` принимает задачу, декомпозирует её и проводит через детерминированный конвейер из 11 специализированных агентов:

```
Explorer → Analyst → Architect → [DevOps: префлайт] → Coder(s) → Tester → Guardian → [Team Lead: решение] → Reviewer
```

Ключевые свойства:

- **Внешняя верификация** — успех определяется реальными тестами (`tester`) и независимым аудитом (`guardian`), а не самооценкой исполнителя.
- **Завершаемость цикла** — лимит циклов доработки, детекция плато и инцидент-протокол исключают бесконечное вращение и неконтролируемый расход токенов.
- **Жёсткая least-privilege модель прав** — права привязаны к ролям и путям на уровне движка: чистые роли не исполняют и не правят код, аудитор не правит код, тестер не может «подогнать» исходник под зелёные тесты, разрушительные и сетевые bash-команды в денилисте.
- **Память вне контекста** — трёхслойная модель памяти (процедурная, обратной связи, концептуальная) живёт в файлах, а не в окне контекста.

Конфигурация разделена на переносимый **глобальный** слой (роли, навыки, общие правила и хардненинг) и **проектный** слой (пути исходников, LSP, контекст конкретного репозитория), который адаптируется под каждый проект — вручную или автоматически через агента `ai-engineer`.

Подробности, развёртывание и адаптация — в [`development-team/README.md`](development-team/README.md).

### 2. OSINT-аналитик (`osint-analyst/`)

Одноагентная система для разведки на основе открытых источников (OSINT) — due diligence, конкурентная разведка, верификация. Агент `osint-analyst` работает по строгой методологии с обязательными точками контроля человека:

- **Трёхуровневая методология** White / Grey / Black с протоколом эскалации Grey-методов через подтверждение пользователя (Human-in-the-Loop).
- **Planning Gate** — перед запуском любых инструментов агент показывает план расследования и ждёт подтверждения.
- **Региональная специализация** — СНГ (РФ, РБ, Украина, Казахстан) и Европа (ЕС/EEA/UK): госреестры, соцсети, поисковые операторы и правовые рамки (ФЗ-152, 99-З РБ, GDPR).
- **Богатый инструментарий** — 13 навыков с инструментами сетевой разведки (Shodan, Censys, theHarvester, SpiderFoot), идентификации и утечек (Sherlock, HIBP, Holehe), анализа метаданных, госреестров СНГ, фреймворка sn0int и готовых Python-шаблонов автоматизации.
- **Профессиональная отчётность** — итоговые отчёты в markdown с диаграммами Mermaid (граф связей, карта инфраструктуры, таймлайн) и гейтом полноты данных.

Конфигурация и навыки — в каталоге [`osint-analyst/`](osint-analyst/) (`opencode.json`, `agents/`, `skills/`).

## С каким агентским ПО применять

Обе системы написаны под **[opencode](https://opencode.ai)** — открытый агентский CLI/TUI для терминала. Формат конфигурации это отражает напрямую:

- `opencode.json` — модели, агенты и модель прав (`permission`);
- `agents/*.md` — определения агентов (front-matter + системный промпт);
- `skills/*/SKILL.md` — навыки в формате skill-файлов;
- `AGENTS.md` — общие правила и контекст проекта.

**Требования:** актуальная версия opencode (модель прав опирается на glob-шаблоны путей в `edit`/`read` и шаблоны команд в `bash` — убедитесь, что версия их поддерживает) и настроенный провайдер моделей. Роли используют модели `ollama-cloud/*`; их можно заменить на любые доступные через ваш провайдер. Команде разработки дополнительно нужен Node.js для LSP-серверов.

Установка opencode и общая схема развёртывания:

```bash
npm install -g opencode-ai@latest
# либо: curl -fsSL https://opencode.ai/install | bash
opencode --version
```

Дальнейшие шаги по каждой системе — в их README/конфигах.

> Формат навыков (`SKILL.md`) и определений агентов близок к экосистеме Claude Code / Claude Agent SDK, поэтому навыки и промпты переносимы как основа; однако модель прав (`permission` с привязкой к путям и bash-денилистами) специфична для opencode и при переносе на другое ПО требует адаптации под его механизм ограничений.
