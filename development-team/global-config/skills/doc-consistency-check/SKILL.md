Применяй как последний шаг перед финальным отчётом.
Документация должна быть согласована — ты последний кто
это проверяет перед тем как задача считается завершённой.

## Что проверить

### 0. Дубли документации (BLOCKER)
Каждому исходному файлу должен соответствовать РОВНО ОДИН docs-файл,
именованный по правилу `canonical_path` (путь с `/` → `-`, без расширения).

Найди случаи, когда один исходный файл описан в двух и более docs-файлах
(например, `routes.py` в `docs/full-project-routes-note.md` и
`docs/src-api-users-routes.md`). Также фиксируй docs-файлы, чьё имя
не соответствует `canonical_path` ни одного существующего исходного файла.

Дубль или осиротевший файл — это BLOCKER, а не INFO: документация
разрастается именно так. Зафиксируй в блоке проблем и, если есть
очевидный канонический файл, укажи какой оставить.

### 1. Полнота покрытия
Все модули изменённые в этой задаче (из DEV-*.md) имеют
актуальные файлы в `docs/`, именованные по `canonical_path`.

### 2. Согласованность имён
Одни и те же модули, функции, классы называются одинаково
во всех файлах `docs/`. Нет синонимов, нет устаревших имён.

### 3. Согласованность связей
Если `module-a` в своём docs-файле указывает `used_by: module-b`,
то `module-b` в своём docs-файле указывает `depends_on: module-a`.
Связи должны быть двусторонними.

### 4. Диаграммы
- Все Mermaid-блоки синтаксически корректны
- Диаграммы не ссылаются на модули которых нет в проекте
- Архитектурная диаграмма в `architecture-note.md` включает
  все модули затронутые задачей

### 5. Даты и задачи
Все обновлённые файлы имеют актуальную дату и номер задачи в заголовке.

## Что делать при нахождении проблем

**Мелкие расхождения** (имя, дата, опечатка) → исправь сам.

**Серьёзные расхождения** (неверные связи, устаревшая архитектура,
отсутствует целый модуль) → зафиксируй в отчёте Reviewer отдельным блоком:

```md
## Проблемы документации

| Файл | Проблема | Статус |
|------|----------|--------|
| docs/full-project-X-note.md | Описание проблемы | Исправлено / Требует внимания |
```

## Self-check

- [ ] Нет дублей: один исходный файл = один docs-файл
- [ ] Нет осиротевших docs-файлов без соответствующего исходного файла
- [ ] Все модули из DEV-*.md покрыты документацией
- [ ] Имена согласованы между файлами
- [ ] Связи двусторонние
- [ ] Диаграммы корректны
- [ ] Даты и номера задач актуальны
