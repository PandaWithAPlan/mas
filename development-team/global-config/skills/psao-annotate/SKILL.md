Применяй, когда агент передаёт сложную инструкцию другому агенту или формирует критически важный выходной артефакт. Навык оборачивает исходный промпт в структуру PSAO (Prompt Segmentation and Annotation Optimisation), добавляя слой управляющих аннотаций без переписывания исходного текста.

# Кто использует

| Агент | Когда |
|-------|-------|
| `team-lead` | При делегировании задач сложности ≥5, REWORK-циклах, или задачах с критическими ограничениями |
| `architect` | При формировании разделов design.md, где критично сохранение контрактов и ограничений |
| `analyst` | При формулировке требований с жёсткими DoD и граничными условиями |
| Любой агент | При передаче инструкции подчинённому агенту с высоким риском семантического дрейфа |

# Концепция

PSAO (Zhang et al., 2024) — метод структурного управления вниманием LLM через три шага:
1. **Сегментация** — разбивка текста на смысловые блоки
2. **Аннотирование** — добавление мягких директив (soft directives) в квадратных `[]` или фигурных `{}` скобках
3. **Инструктивное предложение** — `<meta>`-блок, объясняющий модели как интерпретировать аннотации

**Ключевое свойство:** PSAO никогда не переписывает исходный текст. Он добавляет слой метаданных, сохраняя исходный интент пользователя. Это предотвращает семантический дрейф — главную причину потери точности при передаче задач между агентами.

**Эмпирический эффект:** точность GPT-4o на сложных логических задачах вырастает с 3/10 до 8/10 (50 процентных пунктов аплифта) только за счёт структурирования промпта без изменения модели.

# Словарь аннотаций (Vocabulary V)

## Типы аннотаций

| Тип | Формат | Назначение |
|-----|--------|------------|
| **Важность** | `[критически важно]`, `[очень важно]`, `[важно]` | Указывает относительный вес сегмента для результата |
| **Приоритет** | `[высокий приоритет]`, `[средний приоритет]` | Определяет порядок обработки информации |
| **Контекст** | `[контекст: X]` | Поясняет скрытые смыслы, мотивацию или дополнительные вводные |
| **Намерение** | `[намерение: X]` | Объясняет цель конкретного шага или желаемый эффект |
| **Стиль** | `[стиль: X]` | Задаёт стилистическое требование к конкретному сегменту |

## Правила синтаксиса

1. **Скобки:** использовать квадратные `[]` — эмпирически наиболее эффективный маркер метаданных. Фигурные `{}` допустимы как альтернатива.
2. **Позиционирование:** **суффиксное** — аннотация идёт сразу ПОСЛЕ сегмента, к которому относится. Префиксное (`[важно] сегмент`) слабее.
3. **Структурные теги:** использовать XML-подобные теги `<meta>`, `<задача>`, `<контекст>`, `<ограничения>`, `<ожидаемый_результат>` для сегментации верхнего уровня.

# Процедура

## Шаг 1. Прими вход

Вызывающий агент передаёт:

```
original_prompt: "исходный текст промпта"
intent: "одно предложение — что должно быть достигнуто"
critical_constraints: [
  "ограничение 1 — которое нельзя потерять",
  "ограничение 2",
  ...
] | []
complexity: 1-10  # оценка сложности задачи (опционально)
is_rework: boolean  # повторный цикл?
```

## Шаг 2. Сегментируй (SEGMENT)

Разбей `original_prompt` на смысловые блоки. Используй стандартные разделители:
- **Пустая строка** — граница между логическими секциями
- **Маркеры списков** — каждый пункт списка = отдельный сегмент
- **Ключевая фраза** — если содержит «НО», «ОДНАКО», «КРОМЕ», «ТОЛЬКО ЕСЛИ» — выдели в отдельный сегмент

Правила сегментации:
- Если промпт короткий и однозначный (≤3 предложений без вложенной логики) — НЕ сегментируй, перейди к Шагу 3 только с аннотациями верхнего уровня
- Если содержит условия и исключения — сегментируй обязательно
- Каждый сегмент должен быть осмысленной лингвистической единицей (законченное предложение или фраза)

Результат: `segments = [s1, s2, ..., sn]`

## Шаг 3. Аннотируй (ANNOTATE)

Для каждого сегмента назначь одну или две аннотации по правилам:

| Ситуация | Аннотация |
|----------|-----------|
| Условие, без которого задача не имеет смысла | `[критически важно]` |
| Ключевое бизнес-требование или главная цель | `[очень важно]` |
| Важное, но не блокирующее ограничение | `[важно]` |
| Справочная информация, предыстория, мотивация | `[контекст: ...]` |
| Ограничение на способ решения, инструменты, подход | `[намерение: ...]` |
| Требование к стилю, формату, тону | `[стиль: ...]` |
| Шаг, который надо выполнить до других | `[высокий приоритет]` |

**Правило минимальности:** аннотируй только то, что модель может потерять при чтении длинного промпта. Не аннотируй каждое предложение — это размывает эффект. Цель: 2-5 аннотаций на весь промпт.

**Точка отказа:** если аннотаций >7 — ты переразметил. Вернись и убери наименее критичные.

## Шаг 4. Сформируй PSAO-промпт

### 4.1. Инструктивное предложение (обязательно)

Начни с `<meta>`-блока. Это «локальный системный промпт», объясняющий модели как читать аннотации:

```xml
<meta>
Следуй уровням важности и приоритета в квадратных скобках [] при выполнении задачи.
Аннотации являются управляющими сигналами:
- [критически важно] — условие обязательно к исполнению, блокирующее при нарушении
- [очень важно] — ключевое требование, влияющее на результат
- [важно] — значимое ограничение
- [контекст: ...] — пояснение мотивации или предыстории
</meta>
```

Для делегирования между агентами (Team Lead → подчинённый) используй расширенный мета-блок со спецификой агентной системы.

### 4.2. Структура (оборачивание)

```xml
<meta>
[инструктивное предложение]
</meta>

<задача>
[сегмент 1][аннотация]
[сегмент 2][аннотация]
...
</задача>

<контекст>
[справочная информация][контекст: ...]
</контекст>

<ограничения>
[ограничение 1][критически важно]
[ограничение 2][важно]
</ограничения>

<ожидаемый_результат>
[описание результата и критерии приёмки]
</ожидаемый_результат>
```

### 4.3. Упрощённый режим (plain text)

Если вызывающий агент не использует XML (чат-сообщение делегации), используй упрощённый формат — только суффиксные аннотации `[]` без структурных тегов:

```
[исходный текст сегмента 1][критически важно]
[исходный текст сегмента 2][важно]
[исходный текст сегмента 3][контекст: ...]
```

В этом случае инструктивное предложение вставь первой строкой: «Следуй уровням важности в квадратных скобках []».

## Шаг 5. Верни результат

```
{
  psao_prompt: "полный текст PSAO-аннотированного промпта",
  segments_count: N,
  annotations_applied: [
    { segment: "текст сегмента", annotation: "[критически важно]", reason: "блокирующее условие" },
    ...
  ],
  format: "xml" | "plain",
  instruction_sentence_added: true,
  warnings: [
    "сегментов >10 — рассмотри укрупнение" | "аннотаций нет — промпт слишком простой" | null
  ]
}
```

# Ограничения

- **Не переписывай** исходный текст сегментов — только окружай структурой
- **Не аннотируй всё подряд** — цель 2-5 аннотаций на промпт, иначе эффект размывается
- **Не применяй** PSAO к промптам длиной ≤2 предложения без вложенных условий — это избыточно
- **Не заменяй** явные инструкции аннотациями — `[важно]` не заменяет конкретное «удали дубликаты по полю email»
- **Инструктивное предложение обязательно** — без него аннотации игнорируются моделью

# Self-check

- [ ] Инструктивное предложение (`<meta>` или эквивалент) присутствует
- [ ] Аннотаций 2-5 на весь промпт (не больше 7)
- [ ] Исходный текст НЕ переписан — только добавлены аннотации
- [ ] Критические условия помечены `[критически важно]`
- [ ] Контекстная информация помечена `[контекст: ...]`, а не `[важно]`
- [ ] Суффиксное позиционирование (аннотация ПОСЛЕ сегмента)
- [ ] Формат вывода соответствует входному контексту (xml для файлов, plain для сообщений)
